Things I've Built

01
Rideflow Saga Choreography
Ride booking backend built with six independent microservices communicating exclusively via Kafka events, no direct service-to-service calls. Implements choreography-based Saga pattern where each service reacts to events and handles its own compensating rollback on failure. Driver matching uses Redis geospatial indexing to find the nearest available driver in milliseconds. Real-time GPS tracking streams driver location to the rider via WebSocket and Redis pub/sub. Payments are idempotency-keyed to prevent double charging on duplicate Kafka delivery. Guaranteed event delivery achieved through the Outbox pattern, events survive service crashes. Includes Dead Letter Queue with exponential backoff retry for failed notifications.
